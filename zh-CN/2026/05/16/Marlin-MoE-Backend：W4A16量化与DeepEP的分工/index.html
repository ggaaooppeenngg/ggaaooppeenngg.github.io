背景：MoE 推理的两个瓶颈

在大模型推理中，MoE（Mixture of Experts）架构面临两个独立的性能瓶颈：

1. 计算瓶颈 — 专家权重太大，算不过来

DeepSeek-V3/V4 的 routed expert 权重：

生成阶段 batch=1，每个 token 只激活 6 个专家：

问题：计算量小，但权重大，memory-bound。

2. 通信瓶颈 — 专家分布在多卡，需要 All-to-All

问题：跨卡通信延迟高，占 MoE 总时间 77%。

这两个瓶颈分别由 Marlin（解决计算）和 DeepEP（解决通信）来优化。

Marlin 是什么

Marlin 是 W4A16 量化 kernel：

  • Weight (W): INT4（4-bit 量化）
  • Activation (A): FP16（16-bit 浮点，保持精度）

GitHub: https://github.com/IST-DASLab/marlin
论文: arXiv:2408.11743

核心特点：

  • 针对 小 batch（1-32 tokens） 优化，正好契合生成阶段
  • 理论上 4x 内存压缩（INT4 vs FP16），实际达到 ~3.5x 加速
  • 已集成进 vLLM / SGLang 作为 MoE 后端的量化方案

三、Marlin vs DeepEP：分工明确

维度 Marlin DeepEP
优化目标 计算（GEMM kernel） 通信（All-to-All）
量化策略 W4A16（权重 INT4） FP8（dispatch）+ BF16（combine）
瓶颈类型 Memory-bound（读权重） Communication-bound（跨卡传输）
作用阶段 Expert 计算（本地） Expert 路由（跨卡）
集成位置 vLLM fused_moe kernel EP 通信库（独立）
硬件利用 Tensor Core (INT4) + 异步拷贝 NCCL Gin + NVLink/RDMA

四、完整 MoE 流程：Marlin + DeepEP 协作

数据流：

  1. Router 选出 topk=6 个专家
  2. DeepEP dispatch：token 发给对应卡上的专家
  3. Marlin GEMM：在本地用 W4A16 算 expert（权重小 4x）
  4. DeepEP combine：结果收回原始卡

六、为什么这两个优化不冲突

Marlin 优化 “算”：

  • 权重压缩 4x → HBM 读取量降 4x
  • 针对小 batch（1-32 tokens），生成阶段完美契合
  • 本地计算，不涉及跨卡

DeepEP 优化 “传”：

  • All-to-All 通信优化 → 延迟降 1.7x
  • Hybrid dispatch（NVLink + RDMA）→ 跨节点无压力
  • 0-SM 通信 → SM 全力算 GEMM

七、针对 Hopper (SM90) 的演进

Marlin 原始实现针对 **Ampere (SM80)**，使用 mma.sync（warp 级）和 cp.async（warp 级）。

Hopper (SM90) 的新特性：
| 特性 | Ampere (SM80) | Hopper (SM90) |
|——|——————|—————–|
| Tensor Core | mma.sync (warp 级) | wgmma.mma_async (warpgroup 级，128 threads) |
| 异步拷贝 | cp.async (16B/次) | cp.async.bulk (128B/次) + TMA |
| FP8 支持 | 无 | FP8 Tensor Core (FP8×FP8→FP16) |

Hopper 上的对应实现：DeepGEMM

DeepSeek 的 DeepGEMM（https://github.com/deepseek-ai/DeepGEMM）是 Hopper 上的等价实现：

  • 使用 wgmma.mma_async（warpgroup 级，更大 tile 64×8×16）
  • 支持原生 FP8（无需 dequant，0 开销）
  • TMA 硬件单元直接搬移矩阵块（不占线程）

总结

库 优化目标 核心技术 加速比
Marlin Expert 计算 W4A16 量化 + mma.sync + 双缓冲 Pipeline ~4x（计算）
DeepEP Expert 通信 JIT + NCCL Gin + Hybrid dispatch ~1.7x（通信）
两者结合 端到端 MoE 量化 + 通信协同 ~2.7x（端到端）

选择建议：

  • 只跑单卡 / 小 EP：用 Marlin 足够（W4A16 省显存 + 加速）
  • **跑 EP=8+**：Marlin + DeepEP 一起上，效果叠加
  • **Hopper (H100/H200)**：用 DeepGEMM（FP8）+ DeepEP（FP8 dispatch）获得最佳性能
