иҜӯд№үпјҡAttention/FFN еҗҗеҮәжқҘзҡ„еҚ•и·Ҝз»“жһң x_outпјҢиҰҒе№ҝж’ӯеӣһ 4 и·Ҝ residualпјҢз¬¬ k и·Ҝз”ЁеўһзӣҠ post[n, k] зј©ж”ҫдёҖд»ҪгҖӮ

  • иҢғеӣҙпјҡ(0, 2)пјҲеӣ дёә 2 * sigmoidпјү
  • е…Ғи®ёж”ҫеӨ§пјҲ>1пјүд№ҹе…Ғи®ёиЎ°еҮҸпјҲ<1пјү
  • жҜҸи·ҜзӢ¬з«ӢвҖ”вҖ”з¬¬ 0 и·ҜеҸҜд»ҘжӢҝеҲ° 1.8Г— зҡ„ Attn з»“жһңпјҢз¬¬ 3 и·ҜеҸҜиғҪеҸӘжӢҝ 0.3Г—
  • иҝҷжҳҜ multi-head attention йӮЈз§ҚгҖҢper-head learnable gainгҖҚзҡ„иҝҗиЎҢж—¶еҠЁжҖҒзүҲжң¬

еҜ№еә” model.py з¬¬ 851 иЎҢ post_layer_mix.unsqueeze(-1) * x иҝҷдёҖйЎ№гҖӮ

comb [N, hc=4, hc=4] вҖ” cross-stream routing зҹ©йҳөпјҲ4 и·Ҝ вҶ’ 4 и·Ҝпјү

[\mathrm{out_stream}[n, j, :] \mathrel{+}= \sum_{i=0}^{hc-1} \mathrm{comb}[n, i, j] \cdot \mathrm{residual}[n, i, :]]

иҜӯд№үпјҡеңЁ Attn йӮЈжқЎдё»и·Ҝеҫ„д№ӢеӨ–пјҢ4 и·Ҝ input residual иҝҳиҰҒзӣҙжҺҘи·Ҝз”ұеҲ° 4 и·Ҝ output residualпјҢи·Ҝз”ұзі»ж•°е°ұжҳҜ comb[n, :, :] иҝҷдёӘ 4Г—4 зҹ©йҳөгҖӮ

  • е…ій”®зәҰжқҹпјҡз»ҸиҝҮ Sinkhorn 20 ж¬Ўиҝӯд»Ј вҶ’ еҸҢйҡҸжңәзҹ©йҳөпјҲиЎҢе’Ң=1пјҢеҲ—е’Ң=1пјү
  • еҸҢйҡҸжңәж„Ҹе‘ізқҖгҖҢдҝЎжҒҜе®ҲжҒ’гҖҚпјҡжҜҸи·Ҝ output ж”¶еҲ°зҡ„жҖ»жқғйҮҚ = 1пјҢжҜҸи·Ҝ input иҙЎзҢ®зҡ„жҖ»жқғйҮҚд№ҹ = 1
  • зү©зҗҶж„Ҹд№үзұ»дјјжңҖдјҳдј иҫ“пјҲOptimal TransportпјүпјҡжҠҠ 4 и·Ҝиҫ“е…Ҙзҡ„гҖҢиҙЁйҮҸгҖҚж— жҚҹйҮҚз»„жҲҗ 4 и·Ҝиҫ“еҮә
  • жҜҸдёӘ token иҮӘе·ұеҶіе®ҡдёҖдёӘ 4Г—4 и·Ҝз”ұзҹ©йҳө

еҜ№еә” model.py з¬¬ 851 иЎҢ torch.sum(comb_res_mix.unsqueeze(-1) * residual.unsqueeze(-2), dim=2) иҝҷдёҖйЎ№гҖӮ

дёүиҖ…з»„еҗҲ = е®Ңж•ҙ mHC дёҖеұӮ

дёҖеҸҘиҜқжҖ»з»“пјҡpre еҶіе®ҡгҖҢеҪ“еүҚ token жӣҙдҫқиө–е“Әи·ҜеҺҶеҸІгҖҚпјҢpost еҶіе®ҡгҖҢеҪ“еүҚеұӮиҫ“еҮәеҜ№еҗ„и·Ҝзҡ„йҮҚиҰҒзЁӢеәҰгҖҚпјҢcomb еҶіе®ҡгҖҢеҗ„и·ҜеҺҶеҸІд№Ӣй—ҙжҖҺд№ҲдәӨеҸүж··еҗҲгҖҚвҖ”вҖ”дёүиҖ…йғҪжҳҜжҜҸдёӘ token еҠЁжҖҒз”ҹжҲҗзҡ„гҖӮ

дёәд»Җд№ҲеҸӘжңү comb з”Ё SinkhornпјҢиҖҢ pre / post з”Ё sigmoidпјҹ

еј йҮҸ еҪўзҠ¶ ж•°еӯҰзәҰжқҹ жҝҖжҙ»еҮҪж•° иҢғеӣҙ
pre [N, 4] йқһиҙҹеҚіеҸҜпјҲдёӢжёё Attention/FFN жңүиҮӘе·ұзҡ„ RMSNorm е…ңеә•пјү sigmoid + eps (eps, 1+eps)
post [N, 4] е…Ғи®ёж”ҫеӨ§/иЎ°еҮҸпјҢж— еҪ’дёҖеҢ–иҰҒжұӮ 2 * sigmoid (0, 2)
comb [N, 4, 4] еҝ…йЎ»еҸҢйҡҸжңәвҖ”вҖ”еҗҰеҲҷдҝЎжҒҜдјҡиЎ°еҮҸжҲ–зҲҶзӮёпјҢ4 и·Ҝ residual ж— жі•зЁіе®ҡ cascade Sinkhorn 20 ж¬Ўиҝӯд»Ј иЎҢе’Ң=еҲ—е’Ң=1

comb жҳҜе”ҜдёҖзҡ„гҖҢеӨҡи·Ҝиҫ“е…Ҙ вҶ” еӨҡи·Ҝиҫ“еҮәгҖҚи·Ҝз”ұзҹ©йҳөгҖӮеҰӮжһңдёҚеҪ’дёҖеҢ–пјҡ

  • иЎҢе’Ң < 1 вҶ’ жҹҗи·Ҝ output дҝЎеҸ·ејұеҢ–пјҲж®Ӣе·®иЎ°еҮҸпјү
  • иЎҢе’Ң > 1 вҶ’ жҹҗи·Ҝ output дҝЎеҸ·зҲҶзӮёпјҲж•°еҖјдёҚзЁіе®ҡпјү
  • еҲ—е’ҢдёҚе№іиЎЎ вҶ’ жҹҗи·Ҝ input иў«еӨҡдёӘ output йҮҚеӨҚдҪҝз”ЁпјҢеҸҳзӣёеҠ жқғпјҢзұ»дјј attention зҡ„ mode collapse

Sinkhorn з”Ё 20 ж¬Ўиҝӯд»ЈжҠҠиҝҷдёӘ 4Г—4 зҹ©йҳөжҺЁеҲ°гҖҢиЎҢе’Ң=еҲ—е’Ң=1гҖҚпјҢзӯүд»·дәҺзҰ»ж•ЈжңҖдјҳдј иҫ“пјҲOptimal TransportпјүпјҡжҠҠ 4 дёӘ input ж§ҪдҪҚзҡ„дҝЎжҒҜж— жҚҹеҲҶжөҒеҲ° 4 дёӘ output ж§ҪдҪҚгҖӮ

ж•ҙеј иҜӯд№үеӣҫпјҡpre / post / comb з»„еҗҲ = е®Ңж•ҙ mHC дёҖеұӮ

дёҖеҸҘиҜқжҖ»з»“пјҡ

еј йҮҸ еҪўзҠ¶ дёҖеҸҘиҜқ
pre [N, 4] гҖҢиҝҷдёӘ tokenпјҢжҲ‘иҰҒд»Һ 4 и·Ҝ residual йҮҢеҗ¬и°Ғзҡ„жӣҙеӨҡдёҖзӮ№гҖҚвҖ”вҖ”collapse жқғйҮҚ
post [N, 4] гҖҢAttn з»“жһңиҰҒе№ҝж’ӯз»ҷ 4 и·ҜпјҢжҜҸи·Ҝж”ҫеӨ§еӨҡе°‘гҖҚвҖ”вҖ”per-stream еўһзӣҠ
comb [N, 4, 4] гҖҢ4 и·Ҝ input жҖҺд№Ҳжҙ—зүҢжҲҗ 4 и·Ҝ outputгҖҚвҖ”вҖ”Sinkhorn еҸҢйҡҸжңәи·Ҝз”ұзҹ©йҳө

ж•ҙдёӘ mHC жЁЎеқ—йқ иҝҷ 24 з»ҙпјҲ4 + 4 + 16пјүзҡ„ per-token еҠЁжҖҒеҸӮж•°пјҢи®©жЁЎеһӢеңЁдёҚеўһеҠ  Attn/FFN жқғйҮҚзҡ„еүҚжҸҗдёӢпјҢжҠҠж®Ӣе·®йҖҡйҒ“ж•°д»Һ 1 жӢ“еұ•еҲ° 4пјҢеҗҢж—¶дҝқз•ҷзЁіе®ҡжҖ§е’ҢдҝЎжҒҜе®ҲжҒ’гҖӮиҝҷд№ҹжҳҜдёәд»Җд№Ҳ mHC жҜ”з®ҖеҚ•зҡ„гҖҢ4Г— йҮҚеӨҚеҗҢдёҖдёӘеұӮгҖҚж•ҲжһңеҘҪвҖ”вҖ”еӣ дёә cascade жҳҜ input-aware зҡ„гҖҒжңүеҸҜеӯҰд№ зҡ„ routingпјҢдёҚжҳҜжңәжў°еӨҚеҲ¶гҖӮ

pre / post / comb жҳҜжҝҖжҙ»еҖјпјҲactivationsпјүпјҢдёҚжҳҜжқғйҮҚпјҒ

жҜҸдёӘ forward pass зҡ„жҜҸдёӘ token йғҪиҰҒйҮҚж–°з®—дёҖйҒҚвҖ”вҖ”е®ғд»¬д»Һиҫ“е…Ҙ x з»ҸиҝҮ GEMM + йқһзәҝжҖ§жҙҫз”ҹеҮәжқҘпјҢдёҚеңЁ checkpoint йҮҢпјҢд№ҹдёҚеҸӮдёҺжўҜеәҰжӣҙж–°дҪңдёәеҸӮж•°гҖӮ

зңӢ Block.init йҮҢиў«еЈ°жҳҺдёә nn.Parameter зҡ„жүҚжҳҜжқғйҮҚпјҢиҝҷеұӮеҸӘжңү 6 дёӘпјҡ

и®әж–Үз¬ҰеҸ· д»Јз ҒеҸҳйҮҸ еҗ«д№ү жҳҜеҗҰжқғйҮҚ и®Ўз®—ж–№ејҸ

2. дҝЎжҒҜеҲҶзҰ»еӯҳеӮЁ

3. еҠЁжҖҒе®№йҮҸеҲҶй…Қ

  • еӨҚжқӮ tokenпјҲйңҖиҰҒж·ұеәҰжҺЁзҗҶпјүпјҡB еӨ§е№…ж··еҗҲпјҢи®©жӣҙеӨҡеұӮзҡ„дҝЎжҒҜеҸӮдёҺ
